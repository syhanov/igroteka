# Design Note M01: Выбор структуры landmarks главной страницы

## Статус

Принято.

## Контекст

При разработке главной страницы необходимо выбрать семантическую структуру HTML5 (landmarks), чтобы документ был понятен как пользователям, так и вспомогательным технологиям. Основной задачей было разделить страницу на логические области и использовать элементы по их назначению.

## Решение

Для главной страницы выбрана следующая структура:

- `nav` — панель навигации;
- `section` — блок с интро;
- `section` — блок с лучшими видеоиграми;
- `section` — блок со всеми видеоиграми;
- `article` — карточка каждой игры;
- `footer` — нижняя часть страницы.

### Навигация (`nav`)

Навигационная панель оформлена с помощью тега `nav`, поскольку она содержит ссылки для перехода по разделам сайта и является самостоятельным навигационным блоком страницы.

### Интро (`section`)

Блок с вводной информацией оформлен как `section`, так как представляет собой отдельный тематический раздел главной страницы.

### Списки игр (`section`)

Разделы «Лучшие видеоигры» и «Все видеоигры» также оформлены как `section`, поскольку каждый из них объединяет контент по общей теме и имеет собственный заголовок.

### Карточки игр (`article`)

Каждая карточка игры размечена тегом `article`, поскольку представляет собой самостоятельную единицу контента: содержит название, описание, оценку и может рассматриваться отдельно от остальных карточек.

### Подвал (`footer`)

Нижняя часть страницы оформлена тегом `footer`, поскольку содержит завершающую информацию сайта.

## Рассмотренные альтернативы

### Использование `menu` вместо `nav`

Вариант с тегом `menu` был отклонён. Элемент `menu` предназначен для представления списка команд и имеет более узкую область применения. Для основной навигации по сайту семантически правильнее использовать `nav`.

### Использование `header` вместо `section` для интро

Вариант с использованием `header` был отклонён. Хотя `header` может содержать вводную информацию, в данном проекте интро представляет собой самостоятельный раздел страницы, а не только заголовочную область. Поэтому использование `section` лучше отражает структуру документа.
